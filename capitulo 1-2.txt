1. Responda brevemente, ¿Qué son los datos? ¿Qué es una red de datos? ¿Qué
es una red convergente?

-¿Qué son los datos?: Es la información que se transfiere por medio de una red de datos.

-¿Qué es una red de datos?: Un red de datos hace referencia a sistemas interconectados de dispositivos tecnológicos 
    que permiten la transferencia de información entre ellos.

-¿Qué es una red convergente?: Hace referencia a la tecnología que permite consolidar redes dispersas en una sola plataforma.

2. Describir las características de las arquitecturas de red: tolerante a fallas, escalabilidad, calidad del servicio y seguridad. 
    Mostrar sus respectivos ejemplos.

-Tolerante a fallas: Esta red limita el impacto de una falla  ya sea de software o de hardware. 
                Este tipo de redes se caracterizan por tener enlaces o rutas redundantes entre el origen y destino del mensaje, con el fin
                de garantizar que los mensajes se puedan enrutar de forma instantánea si algún enlace o ruta falla.
                Se tomará como ejemplo de este ripo de red al internet en sus comienzos ya que este era respaldado por el DoD
                (Departamento de Defensa de Estados Unidos). Este Departamento tenía como objetivo tener un medio de comunicación
                que pudiera soportar la destrucción de numerosos sitios de transmisión sin interumpir el servicio; dicho objetivo lo lograron
                mediante rutas o enlaces redundantes en la red.

-Escalabilidad: Una red que sea escalable tiene la capacidad de expandirese rápidamente para adminitr nuevos usuarios sin afectar el rendimiento
                del servicio. Para que una red tenga esta capacidad debe existir un diseño jerárquico en capas, de esta manera es posible que cada usuario y 
                operador se puedan insertar sin causar conflictos en toda la red. Un ejemplo es el internet actual el cual se expande
                con gran facilidad, todo esto gracias a la función del diseño de los portocolos y tecnologías sobre las cuales está construido
                Internet. Además este cuenta con jerarquía en capas para diferentes servicios lo que permite que los operadores y usuarios 
                mantengan una comunicación directa sin ser interrumpida por otra.

-Calidad del servicio: Con el desarrollo de nuevas aplicaciones se ha visto la necesidad de que transmisiones de voz y video tengan un nivel de 
                        calidad consistente y sin interrumciones. Es por esto que La calidad de estos servicios se mide con la calidad de 
                        experimentar la misma presentación de audio y video en persona. Un ejemplo de esto son los mecanismos para administrar
                        el tráfico en las redes. Ya que la congestión se genera cuando la demanda de recursos de red supera la capacida de esta.
                        Gracias a estos mecanismos se asegura que los paquetes de datos lleguen a tiempo y en orden.

-Seguridad: Un red debe ser segura ya que se intercambia todo tipo de información confidencial. Para lograr esto los administradores pueden proteger la red con seguridad tanto 
            hardware como software para evitar el acceso de terceros a redes privadas con el objetivo de proteger la información. Un ejemplo 
            de redes que necesitan un amplio sistema de seguridad con las organizaciones gubernamentales o bancarias, ya que estas manejan 
            un tráfico bastante importante de información críctica y que si se da un mal tratamiento de estos datos llevaría a daños permanentes
            a las empresas y a los clientes.


3. Investigar qué son las redes orientadas y no orientadas a la conexión.
- REDES ORIENTADAS A LA CONEXIÓN: Son aquellas que ofrecen servicios como el sistema telefónico conmutado por circuito, donde se garantiza la calidad y 
                                consistencia de los mensajes transmitidos. Por otro lado, el proveedor de servicios puede cargar a los usuarios de la red 
                                durante el período de tiempo en que la conexión se encuentra activa.

- REDES NO ORIENTADAS A LA CONEXIÓN: Se envían paquetes individuales que se enrutan de manera independiente. Por tanto, hay una mayor flexibilidad para
                                     enrutar y adaptarse a cambios en la red, y menor sobrecarga ya que no se requiere establecer la conexión.


Se concluye que dependiendo de las necesidades de la aplicación se selecciona entre redes orientadas a la conexión y no orientadas a la conexión. Ya que
la primeras garantizan la entrega y orden de los datos, al contrario de las no orientadas a la conexión que se adecuan a la flexibilidad de las aplicaciones.

                
4. Describa que es la calidad del servicio y responda: ¿Por qué importa la calidad del servicio en una red de datos?

La calidad de servicio hace referencia al rendimiento en la que es capaz un red en entregar datos, es decir, garantizar una buena experiencia al usuario 
independiente del tipo de tráfico en una red. La calidad del servicio importa en una red de datos debido a:

    -  Se debe ofrecer una experiencia fluida sin interrupciones, por ejemplo, las aplicaciones como transmisiones de voz y video en tiempo real que requieren 
    un ancho de banda consistente para dar una buena experiencia al usuario.

    -  Es importante priorizar los paquetes de voz para evitar retrasos, por ejemplo, una red que transporta voz y datos, la cual necesita garantizar una 
    comunicación clara.
    
    -  Asigna eficientemente los recursos de red para que sean usados de manera adecuada en cada tipo de tráfico, de esta manera no se afecta el rendimiento 
    general.

1. Describir la estructura de una red, incluidos los dispositivos, medios y servicios necesarios para lograr comunicaciones exitosas.

Una red se compone de varios elementos que colaboran entre si para permitir que la comunicación se realice de forma exitosa.
Estos componentes incluyen dispositivos, medios y servicios. A continuación, se describe cada uno de estos aspectos:

COMPONENTES DE RED:

-Dispositivos y Medios: Los dispositivos y los medios son los elementos físicos o hardware de la red. Los dispositivos son componentes
                       como computadoras, impresoras, teléfonos VoIP, cámaras de seguridad y dispositivos móviles. 
                       Los medios son los canales a través de los cuales se transmiten los datos, como cables metálicos, fibras ópticas y transmisión inalámbrica (ondas electromagnéticas).

-Servicios y Procesos: Los servicios y procesos son el software que se ejecuta en los dispositivos conectados a la red. 
                      Los servicios son programas de comunicación que proporcionan información en respuesta a solicitudes, como servicios de 
                      correo electrónico y hosting web. Los procesos son funciones que permiten direccionar y trasladar mensajes a través de la red, como la retransmisión 
                      de señales, la administración de rutas y la priorización de mensajes según la calidad de servicio.

Dispositivos finales y su rol: Los dispositivos finales, también llamados hosts, son aquellos con los que las personas interactúan 
                                directamente. Estos dispositivos sirven como interfaz entre la red humana y la red de comunicación subyacente. 
                                Algunos ejemplos incluyen computadoras, impresoras de red, teléfonos VoIP, cámaras de seguridad y dispositivos móviles. 
                                Los dispositivos finales pueden funcionar como clientes, servidores o ambos, según el software instalado en ellos.

Dispositivos intermedios y su rol: Los dispositivos intermedios son aquellos que proporcionan conectividad y gestionan el flujo de 
                                    datos en la red. Estos dispositivos trabajan detrás de escena para garantizar que los datos fluyan de 
                                    manera eficiente y segura.Algunos ejemplos son:            

                        -Dispositivos de acceso a la red: Hubs, switches y puntos de acceso inalámbricos que conectan dispositivos finales a la red.
                        -Dispositivos de internetworking: Routers que conectan diferentes redes y determinan la ruta óptima para los datos.
                        -Servidores de comunicación y módems: Facilitan la comunicación y la conexión a la red.
                        -Dispositivos de seguridad: Firewalls que protegen la red de amenazas externas.

Medios de Red: Los medios de red son los canales a través de los cuales se transmiten los datos. Los principales tipos de medios son:

        -Hilos metálicos: Utilizan cables con impulsos eléctricos para transmitir datos.
        -Fibras ópticas: Emplean pulsos de luz para transmitir datos a través de fibras de vidrio o plástico.
        -Transmisión inalámbrica: Utiliza ondas electromagnéticas para la transmisión, como en redes Wi-Fi y comunicación satelital.

2. Comparar y contrastar los siguientes términos: RED, LAN, WAN, INTERNETWORK E INTERNET.

Red:
    -Una red es un conjunto de dispositivos interconectados que pueden comunicarse y compartir recursos.
    -Puede abarcar desde una pequeña red local hasta una red global interconectada, como Internet.
    -El término "red" es un concepto general que engloba diversos tipos de redes, como LAN, WAN e INTERNET.

LAN (Red de Área Local):
    -Una LAN es una red de pequeña escala que cubre un área geográfica limitada, como un edificio o un campus.
    -Administrada por una única organización y controlada por políticas internas de seguridad y acceso.
    -Conexiones de alta velocidad y baja latencia, ideal para compartir recursos locales.

WAN (Red de Área Amplia):
    -Una WAN abarca distancias geográficas más amplias y conecta múltiples LAN a través de proveedores de servicios de telecomunicaciones.
    -Permite la comunicación entre ubicaciones geográficamente dispersas.
    -Control administrativo dividido entre la organización y el proveedor de servicios de telecomunicaciones.

Internet:
    -Internet es una red global de redes interconectadas que permite la comunicación a nivel mundial.
    -Está compuesta por redes de proveedores de servicios de Internet (ISP) interconectadas.
    -Accesible para millones de usuarios y utiliza tecnologías y protocolos estandarizados para asegurar la comunicación efectiva.

Internetwork:
    -Una internetwork, también conocida como red de redes, es una malla de redes interconectadas que cubre necesidades de comunicación humana.
    -Puede incluir redes privadas y públicas, como las pertenecientes a organizaciones o empresas.
    -Internet es el ejemplo más conocido de una internetwork, formada por la interconexión de redes ISP.

4. Describir la diferencia entre Tarjeta de interfaz de red (NIC), puerto físico e interfaz de red.
- Tarjeta de Interfaz de Red (NIC): Proporciona la interfaz física entre un dispositivo y la red, permite que el dispositivo pueda conectarse a una 
red inalámbrica o cableada.

- Puerto Físico: Conecta el medio de transmisión como cables de fibra óptica o Ethernet. Establecen conexión física entre dispositivos.

- Interfaz de red: Es el puerto que se conecta a una red individual. Es decir, si un router conecta dos redes, tendrá dos interfaces para cada red. 
Las interfaces permiten la comunicación entre redes ya sean conexiones físicas o lógicas.

5. ¿Por qué se dice que un protocolo es independiente de la tecnología?
    Un protocolo se considera independiente de la tecnología porque su función y descripción se centran en qué debe hacerse en una 
    comunicación específica, en lugar de cómo se debe hacer. Esto significa que el protocolo establece las reglas y las funciones que deben 
    seguirse durante la comunicación, pero no prescribe la tecnología, el software o los detalles técnicos específicos 
    que deben emplearse para implementar esas reglas.

6. Consultar y explicar brevemente qué es una Unidad de Datos del Protocolo (PDU).
    
    Una Unidad de Datos del Protocolo (PDU) es una unidad básica de información que se transmite a través de una red utilizando un 
    protocolo de comunicación. Las PDUs son utilizadas para encapsular y transportar datos entre dispositivos en una red, asegurando que 
    los datos sean entregados de manera coherente y ordenada.En pocas palabras, una PDU es una porción de datos que contiene la
    información necesaria para la comunicación en una red. Dependiendo del protocolo y del nivel de la red en el que se encuentre, 
    una PDU puede tener diferentes nombres y estructuras.

9. Describir la función de cada capa en los dos modelos de red: TCP/IP y OSI.


MODELO TCP/IP:

    - APLICACIÓN: Representa datos para el usuario más el control de codificación y de diálogo.
    - TRANSPORTE: Admite la comunicación entre distintos dispositivos de distintas redes.
    - INTERNET: Determina la mejor ruta a través de la red.
    - ACCESO A LA RED: Controla los dispositivos del hardware y los medios que forman la red.

MODELO OSI:

    - APLICACIÓN: Proporciona los medios para la conectividad de extremo a extremo entre individuos que usan redes de datos.
    - PRESENTACIÓN: Representa los datos transferidos entre los servicios de la capa de aplicación.
    - SESIÓN: Proporciona servicios de la capa de Presentación para organizar su diálogo y adminiistrar el intercambio de datos
    - TRANSPORTE: Define los servicios para segmentar los datos para las comunicaciones entre dispositivos finales.
    - RED: Proporciona servicios para intercambiar datos individualas en la red entre dispositivos.
    - ENLACE DE DATOS: Describe los métodos para intercambiar tramas de datos en un medio común.
    - FISICA: Describe los medios para mantener y desactivar conexiones físicas para la transmisión de bits entre dispositivos.
